##2: Write a command that makes a blast database for the database_mystery_proteins.fasta file

makeblastdb -in database_mystery_proteins.fasta -dbtype prot -out prot_database


##3: Write a command that searches the mystery nucleotide query against the mystery
#transcripts database and: a.) keeps sequences with an e-value of 1e-3 or less, b.) keeps up to
#5 hits, and c.) outputs the file to tabular format with comment lines.

blastn -db nuc_database -query query_mystery_nucleotide.fasta -outfmt 7 -evalue 1e-3 -out nuc_blast_results.txt -max_target_seqs 5

 	## -evalue sets evalue maximum, -max_target_seqs specifiies number of hits. -outfmt <string> is output format; 7 means tabular w/ comment lines


##4: What is/are the IDs/sequence names of most similar hit(s) from the above search (based on sequence identity)?

# T010027234997	91.965

##5:  Using awk and/or grep, write a command that prints the sequence names with an identity
#over 90% from the above search to a new file called most_similar.txt.

grep -v "#" nuc_blast_results.txt | awk '$3 >90 {print $2}' >> most_similar.txt

	## -v "#" will get rid of anything that is commented out (so, the headers). grep must go first
	## so awk will be able to find the correct columns
